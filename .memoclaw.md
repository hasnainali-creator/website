# PROJECT MEMOCLAW: OMNYSPORTS
The ultimate memory brain for the OmnySports project.

## User & Identity
- **User**: Hasnain Alli (The mastermind. Sajjad Ahmad is his chachu whose laptop this is).
- **Communication Flow**: Pure **Hinglish** (Roman Urdu/Hindi mixed with English). No formal robotspeak or boring text. Friendly, direct, "dost" style.
- **Vibe expected**: Features must be "Insane Level". Advanced UI/UX, premium feel, modern, highly interactive, fast. No generic designs.

## Technology Stack 🚀
- **Package Manager**: **BUN** ONLY (`bun run dev`, `bun add`, etc.). DO NOT USE NPM/YARN.
- **Framework**: Astro (SSG/SSR optimized).
- **Frontend Components**: React (`.tsx`) and Astro (`.astro`).
- **Styling**: Tailwind CSS **v4**. (Using modern `@layer`, raw CSS vars, and CSS nesting. Not old Tailwind directives).
- **CMS**: Keystatic (Local Markdown/MDX CMS).

## Crucial Preferences & Rules
- **Aesthetic**: Premium, minimalist, institutional. Light-mode strictly enforced (Dark mode has been entirely purged across the app!).
- **Error Handling**: Build errors are unacceptable. Fix TS warnings elegantly.
- **Validation Philosophy**: Real-time checking, animated tooltips, shake animations, visual feedback.

## Active Ongoing Implementation
- **Push Notification System**: 
  - Required stack: Firebase Cloud Messaging (FCM tokens using `firebase-admin`).
  - Expected Logic: Custom elegant prompt > Receive standard device specific push notifications.
  - Automations: Bun script hooked into build or commit hook to automatically read new articles from local/Keystatic folder and push the notification via Firebase topics (`latest_news`) seamlessly across all devices.
